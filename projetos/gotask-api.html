GoTask API

API REST para gerenciamento de tarefas construída em Go, com autenticação via JWT, persistência em PostgreSQL e deploy containerizado com Docker.

Ver projeto Ver código

Visão geral

O GoTask API é um serviço backend que organiza tarefas por usuário, com controle de prazos, status e prioridades. O foco do projeto foi construir uma API limpa, segura e fácil de manter, aplicando boas práticas de arquitetura em Go.

O problema

Aplicações de produtividade precisam de um backend confiável que isole os dados de cada usuário, valide entradas e responda rápido, mesmo sob carga. O desafio era entregar isso com uma base de código organizada, testável e pronta para evoluir.

A solução

Desenvolvi uma API REST em Go seguindo uma separação clara de camadas (handlers, serviços e repositórios). A autenticação usa JWT, os dados ficam em PostgreSQL e toda a aplicação é empacotada em Docker, facilitando o deploy em qualquer ambiente.

Funcionalidades

  • Cadastro e login de usuários com hash de senha
  • Autenticação e autorização por token JWT
  • CRUD completo de tarefas com filtros por status e prazo
  • Validação de entrada e respostas de erro padronizadas
  • Paginação e ordenação nas listagens
  • Healthcheck e logs estruturados

Arquitetura

A estrutura segue o fluxo requisição → handler → serviço → repositório → banco. Os handlers tratam HTTP, os serviços concentram a regra de negócio e os repositórios isolam o acesso ao PostgreSQL — o que torna cada camada testável de forma independente. O container Docker sobe a API junto a um banco PostgreSQL via docker-compose.

Desafios & aprendizados

O principal desafio foi estruturar a autenticação de forma segura sem acoplar o JWT à regra de negócio. Resolvi isso com um middleware dedicado, que valida o token antes da requisição chegar aos serviços. O projeto reforçou meu domínio sobre design de APIs, modelagem relacional e containerização — e como cada decisão de arquitetura impacta a manutenção a longo prazo.
