URLShort

API encurtadora de URLs construída em Go, com cache em Redis para redirecionamentos de baixa latência, persistência em PostgreSQL e deploy containerizado com Docker.

Ver projeto Ver código
⇄ URLShort · go + redis

Visão geral

O URLShort é um serviço backend que transforma URLs longas em links curtos e os resolve com baixa latência. O foco foi unir uma API REST limpa em Go a uma camada de cache eficiente, garantindo redirecionamentos rápidos mesmo sob alto volume.

O problema

Encurtadores de URL precisam responder o redirecionamento em poucos milissegundos e aguentar muitas leituras repetidas do mesmo link. Consultar o banco a cada acesso não escala — era preciso uma estratégia de cache que reduzisse essa carga sem perder consistência.

A solução

Desenvolvi uma API REST em Go que gera códigos curtos únicos e os persiste no PostgreSQL. Os links mais acessados ficam em cache no Redis, de modo que o redirecionamento responde direto da memória. Toda a aplicação é empacotada em Docker para subir em qualquer ambiente.

Funcionalidades

  • Encurtamento de URLs com código curto único
  • Redirecionamento de baixa latência com cache em Redis
  • Persistência dos links e metadados no PostgreSQL
  • Contagem de cliques por link
  • Validação de URL e respostas de erro padronizadas
  • Healthcheck e ambiente reprodutível com Docker Compose

Arquitetura

O fluxo de leitura é requisição → checa Redis → (se faltar) consulta PostgreSQL → popula o cache → redireciona. A escrita grava no PostgreSQL e invalida/atualiza o cache. Cada serviço roda em seu próprio container, comunicando-se por uma rede interna do Docker Compose.

Desafios & aprendizados

O principal desafio foi manter cache e banco coerentes sem comprometer a velocidade. Resolvi isso definindo uma política clara de expiração e invalidação do Redis. O projeto aprofundou meu domínio sobre cache distribuído, design de APIs e otimização de latência.
