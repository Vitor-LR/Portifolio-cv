DataMart SQL

Pipeline de ingestão, limpeza e análise de dados em Python e SQL, gerando relatórios e indicadores a partir de dados armazenados em PostgreSQL.

Ver projeto Ver código

Visão geral

O DataMart SQL pega dados brutos de diferentes fontes, organiza em um banco relacional e extrai indicadores úteis por meio de consultas SQL e análise em Python. Une duas das minhas áreas de foco: dados e backend.

O problema

Dados espalhados em planilhas e arquivos não respondem perguntas de negócio com agilidade. O objetivo era centralizar esses dados, tratá-los e disponibilizá-los de forma consultável e confiável.

A solução

Construí um pipeline em Python (com Pandas) para ler, limpar e padronizar os dados, carregando-os em um modelo relacional no PostgreSQL. Sobre essa base, escrevi consultas SQL otimizadas que geram os relatórios e métricas finais.

Funcionalidades

  • Ingestão de dados de CSV/planilhas com validação
  • Limpeza e normalização com Pandas
  • Modelagem relacional e carga no PostgreSQL
  • Consultas SQL para indicadores e agregações
  • Exportação de relatórios prontos para análise

Arquitetura

O fluxo segue o padrão extração → transformação → carga (ETL): Python lê e trata os dados, popula as tabelas no PostgreSQL, e as consultas SQL ficam isoladas em scripts versionados, fáceis de revisar e reaproveitar.

Screenshots

Desafios & aprendizados

O desafio central foi garantir a qualidade dos dados: tratar valores ausentes, padronizar formatos e evitar duplicidades antes da carga. O projeto consolidou minha base em SQL, modelagem de dados e análise com Python, áreas que pretendo continuar aprofundando.
